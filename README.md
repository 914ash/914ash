# Applied systems portfolio

This portfolio collects geospatial monitoring tools, analyst-facing research software and agentic workflow systems built for work that needs visible state, explicit review paths and reproducible behavior.

The strongest repos show operator-facing maps, planning-to-execution infrastructure and public-safe analysis tools with clear provenance.

## About

The through-line is operational judgment: make state visible, keep interfaces legible and avoid hiding critical decisions behind opaque automation.

These projects support review, triage and structured handoff. They are working systems or reference snapshots, not concept decks.

## Featured projects

### [intelligence-analysis-skill-pack](https://github.com/914ash/intelligence-analysis-skill-pack)

A portable skill pack that teaches AI agents structured analytic tradecraft using public-source methods, references and writing patterns.

It matters because it turns analytical method into a reusable artifact instead of leaving it trapped inside one-off prompts.

### [satellite-challenge](https://github.com/914ash/satellite-challenge)

A maritime and aviation tracking dashboard with replay, shared contracts and entity review. It uses a TypeScript API, a React web app and shared types.

It matters because it is the clearest end-to-end operator interface in the public portfolio.

### [symphony](https://github.com/914ash/symphony)

A TypeScript workflow-service fork for long-running coding-agent execution with visible run state and guarded completion. It uses Node.js, Linear integration and an operator dashboard.

It matters because it focuses on inspectability and safer automation, not just task dispatch.

### [agentic-stories-to-symphony](https://github.com/914ash/agentic-stories-to-symphony)

A spec-first planning layer that captures intent, stores approved revisions and projects deterministic Linear stories before execution.

It matters because it treats planning as a system boundary instead of an informal pre-step.

### [dprk-document-analysis-app](https://github.com/914ash/dprk-document-analysis-app)

A public sanctions-analysis monorepo with analyst-facing dashboards, entity resolution and temporal network-drift tooling.

It matters because method, lineage and reviewer-facing outputs live in the same public surface.
