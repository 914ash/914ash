# Nick Ashley

I build operator-facing software, geospatial interfaces, and AI workflow systems for operational and defense-adjacent use cases.

This profile is a curated front door to the projects that best show how I work: practical software, clear system design, and tools built for decision support rather than demos alone.

## Featured Projects

### [traffic-cams](https://github.com/914ash/traffic-cams)
Map-based traffic camera review application that normalizes public feeds into a single operator surface. Highlights geospatial UI design, realtime ingest cleanup, and a fast inspection workflow.

**Stack:** Node.js, TypeScript, React, WebSockets, map UI

### [satellite-challenge](https://github.com/914ash/satellite-challenge)
Maritime and aviation tracking dashboard with replay, shared API contracts, and entity review. This is the strongest end-to-end operational dashboard in the portfolio.

**Stack:** TypeScript, React, Express, shared contracts, tactical map UI

### [symphony](https://github.com/914ash/symphony)
TypeScript workflow service fork for inspectable, long-running coding-agent execution. Focused on observability, verification-aware completion, and safer autonomous runtime behavior.

**Stack:** TypeScript, Node.js, HTTP dashboard, Linear integration, agent orchestration

### [agentic-stories-to-symphony](https://github.com/914ash/agentic-stories-to-symphony)
Spec-first planning layer that captures requirements, gates approval, projects deterministic Linear stories, and hands structured work into Symphony-style execution.

**Stack:** TypeScript, Node.js, workflow tooling, Linear integration

### [dprk-document-analysis-app](https://github.com/914ash/dprk-document-analysis-app)
Sanctions-intelligence monorepo with analyst-facing dashboards, entity resolution, and temporal network-drift analysis. Built for method visibility, source lineage, and reproducible review.

**Stack:** Python, dashboard tooling, graph analysis, document-processing pipelines
