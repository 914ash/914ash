# Featured Project Notes

## Portfolio Inventory

| Project | Status | What The Project Is | Why It Belongs Here |
| --- | --- | --- | --- |
| [`traffic-cams`](https://github.com/914ash/traffic-cams) | Active prototype | A geospatial monitoring tool that pulls public traffic-camera feeds into one map-based review surface, then lets a reviewer move from regional coverage to camera-level inspection quickly. | It shows practical feed normalization, realtime update handling, and an operator-facing interface instead of a static map demo. |
| [`satellite-challenge`](https://github.com/914ash/satellite-challenge) | Active prototype | A live maritime-and-aviation tracking dashboard with replay controls, shared API contracts, and entity-focused review. | It is the clearest full-stack operations product in the current public set and demonstrates the strongest end-to-end UI story. |
| [`symphony`](https://github.com/914ash/symphony) | Active fork | A workflow-service fork for running issue-driven coding-agent work with visible runtime state, guarded completion, and operator-readable APIs. | It represents the infrastructure side of the portfolio: not just automation, but automation that can be inspected and controlled. |
| [`agentic-stories-to-symphony`](https://github.com/914ash/agentic-stories-to-symphony) | Active fork / planning layer | A planning-to-execution system that captures product intent, stores approved spec revisions, and projects deterministic Linear stories before handoff to execution. | It makes the case that agentic execution should start from explicit planning artifacts, not improvised ticket creation. |
| [`dprk-document-analysis-app`](https://github.com/914ash/dprk-document-analysis-app) | Research / demo | A public sanctions-analysis repo that combines analyst-facing dashboards, entity-resolution workflows, and temporal network-drift analysis with documented source lineage. | It adds research and analysis tooling to the portfolio without hiding method or provenance behind private data. |
| [`intelligence-analysis-skill-pack`](https://github.com/914ash/intelligence-analysis-skill-pack) | Active public pack | An experiment in teaching AI agents structured analytic tradecraft for intelligence-style analysis by packaging methods, references, and writing patterns gathered from public web sources into a reusable skill library. | It shows workflow design as a product in its own right and makes analytical method portable across agents and repos. |
| [`attractor-workflow-demo`](https://github.com/914ash/attractor-workflow-demo) | Demo | A demonstration repo that uses a DOT-defined AI workflow to move from requirements capture through implementation, testing, and browser validation on a small habit-tracker app. | It is useful as a methodology artifact, but narrower and less current than the main featured systems. |
| [`LlamaIndex-RAG-Linux-CUDA`](https://github.com/914ash/LlamaIndex-RAG-Linux-CUDA) | Archived reference | A notebook-based local-RAG experiment for Linux and CUDA environments using Word documents and locally hosted models. | It stays in the portfolio as technical history and environment reference rather than as current product work. |
| [`private-gpt`](https://github.com/914ash/private-gpt) | Reference snapshot | A preserved snapshot of a PrivateGPT-style codebase used for local experimentation with private document QA and RAG workflows. | It is kept for experimentation lineage and comparison, not presented as original upstream authorship. |

## Curation Notes

- `traffic-cams`, `satellite-challenge`, `symphony`, `agentic-stories-to-symphony`, `dprk-document-analysis-app`, and `intelligence-analysis-skill-pack` form the current public core.
- Older demos and reference repos stay visible, but they are intentionally de-emphasized so the portfolio reads as a coherent current body of work rather than a dump of experiments.
- Where public impact evidence is not available, the descriptions stay close to architecture, workflow, and review surface rather than making inflated claims.
